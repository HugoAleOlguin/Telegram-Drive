# Reporting Security Issues

If you discover a security vulnerability in Telegram Drive, please report it privately.

Do not open a public GitHub issue for security problems, as this could expose other users to risk before a fix is available.

## How to Report

Send a description of the issue to the project maintainer via email or a private GitHub security advisory.

Include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Your suggested fix (optional)

## Scope

The following are considered in scope for security reports:

- Exposure of Telegram API credentials (api_id, api_hash, session files)
- Unintended data transmission to third-party servers
- Local privilege escalation
- Vulnerabilities in file encryption implementation

## Our Commitment

We will acknowledge your report within 48 hours and aim to release a fix within 14 days for critical issues.
